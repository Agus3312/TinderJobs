### Coding Standards
- Functional components with TypeScript interfaces (no default exports unless also exporting named).
- `@/` path alias for imports (`@/components`, `@/types`, `@/data`).
- Tailwind CSS v4 for layout/spacing; CSS custom properties for theming (dark-only Zinc palette + emerald accent).
- `"use client"` on all interactive components (App Router SPA-style).
- Inline `style` props for themed colors (`var(--bg)`, `var(--accent)`, etc.) — NO Tailwind color utilities.
- Framer Motion used sparingly (exit animations, spring bottom sheets) — no heavy animations or glassmorphism.
- Lucide React for all icons.
- `Inter` (sans) + `JetBrains Mono` (mono/numbers) font stack.
- Reduced motion media query respected (`prefers-reduced-motion`).
- ESLint 9 (flat config) with next.js defaults. No Prettier.
- TypeScript strict mode enabled.
- No test infrastructure — manual/browser verification only.
